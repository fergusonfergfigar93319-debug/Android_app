## 1. 项目概述

### 1.1 项目名称

**同频搭名片（BuddyCard）**

### 1.2 一句话介绍

用户录入个人偏好后生成专属“同频搭名片”，个性化智能体帮助一键生成招募帖并推荐合拍搭子；建立搭子关系后生成沟通共识与轻复盘建议，提升组队稳定性与体验。

### 1.3 核心目标

- **降低找搭子成本**：更快找到时间、目标、位置、沟通风格匹配的队友  
- **减少沟通摩擦**：通过“共识卡+话术建议”降低冲突、提升协作  
- **提升留存与成长**：轻量训练/复盘沉淀，形成“越用越懂你”的个性化体验

---

## 2. 背景与痛点

### 2.1 行业现象

电竞玩家规模巨大，但稳定的固定队/搭子稀缺。传统“加好友—约时间—打一把—散伙”的临时组队模式，效率低且体验不稳定。

### 2.2 典型痛点

- **匹配低效**：常玩时段不重合、目标不一致（冲分 vs 娱乐）、位置冲突导致无法长期组队  
- **沟通摩擦**：指挥风格不同、情绪化表达、缺少队伍共识与复盘机制  
- **表达成本高**：很多用户不会写招募帖/不会开口，导致招募效果差  
- **关系难沉淀**：缺少“共同目标+协作规则”，组队关系易破裂

---

## 3. 目标用户与场景

### 3.1 目标用户

- **新手/轻度玩家**：希望找到友善且同频的队友，提升体验与上手效率  
- **校园玩家/社团玩家**：希望快速组织开黑/训练赛，建立稳定队伍  
- **碎片时间玩家**：需要能与自己时间高度重合的搭子

### 3.2 核心使用场景

- **赛前找人**：我要今晚 9 点后上分，找同位置互补、愿意语音沟通的队友  
- **论坛招募**：我想发一条招募帖，但不知道怎么写才更容易被回应  
- **组队后磨合**：刚配合两局有分歧，需要一个“共识规则”降低摩擦  
- **赛后复盘（轻量）**：希望得到“下局最值得改的一个点”，避免信息过载

## 4. 解决方案（产品闭环）

### 4.1 闭环链路

1) **建档**：用户录入个人信息与偏好（时间、目标、位置、沟通风格等）  
2) **名片生成**：智能体生成“同频搭名片”（标签+宣言+组队规则）  
3) **找搭子双通路**：  
   - **推荐流**：系统给出 Top N 合拍用户（含推荐理由与冲突提示）  
   - **论坛招募**：智能体一键生成招募帖并发布，吸引同频用户互动  
4) **建立关系**：申请搭子/通过后建立“搭子关系页”  
5) **羁绊沉淀**：生成“共识卡”（沟通公约+共同目标）与轻复盘建议  
6) **优化画像**：根据互动与复盘反馈更新画像，提高后续推荐准确度

### 4.2 论坛（搭子广场）——核心功能之一

- **主 Tab 固定入口**：与「推荐」「我的」并列，产品内称为 **「广场」**，承担 **招募发帖、浏览、搜索与分区** 能力，与推荐流形成 **双通路找搭子**。  
- **闭环角色**：用户可 **AI/手动发帖** → 帖子进入广场列表 → 他人浏览详情 → **申请搭子 / 回帖**（与关系链、名片共用画像）。  
- **详细设计与 Android 实现映射**：见仓库内 **《论坛模块功能设计（核心功能）》.md**（分区枚举、Post 扩展字段、`ForumRepository` 与后续 API 对齐说明）。

### 4.3 个人定制智能体——核心功能之一（**用户设计专属智能体**）

- **与建档的关系**：建档决定长期画像（游戏、时段、沟通风格等）；智能体在画像之上由用户 **命名、选人格基底与角色皮、定音色语义**，并填写可选 **口头禅 / 结束语 / 不要这样说 / 补充人设**；另支持 **无需重新建档** 的实时参数（表达强度、回复长度、场景侧重）。  
- **专属感**：支持 **多套方案（槽位）** 切换（如「冲分严肃」「娱乐整活」），与发帖、推荐解释、模拟对话共用同一套 **可持久化的 Agent 配置**。  
- **产品内入口**：**底栏独立 Tab「创作」**（点击进入 **创作专属智能体**）为主入口；**推荐流顶部 Banner**、**我的页** 主文案 **「点击创作专属智能体」** 进入同一路由；**发帖页** 展示当前智能体摘要（含专属句），强化与「AI 写招募」的闭环感知。  
- **闭环角色**：与「一键生成招募帖」「推荐解释话术」共用 **结构化 Agent Config + 画像** 生成 LLM System 片段；接后端后槽位与微调持久化并下发。  
- **详细说明**：见仓库 **《个人定制智能体功能设计》.md**（V1.1）。

## 5. 核心创新点

### 创新点 1：AI 生成“同频搭名片”

将用户画像转化为可传播、可匹配、可解释的卡片信息：

- **三标签**：时间/目标/风格（例：夜猫子｜冲分党｜稳健运营）  
- **组队宣言**：一句话自我介绍（可选风格：温柔/搞笑/战术派）  
- **组队规则**：3 条可执行约定（例：连跪两把休息 5 分钟；只喷操作不喷人）

### 创新点 2：可解释的匹配推荐

推荐不仅给分数，还输出：

- **推荐理由 3 条**（时间重合、目标一致、位置互补等）  
- **潜在冲突 1 条**（风格差异/语音偏好不同）  
- **解决建议 1 条**（给出具体沟通方案或折中方式）

### 创新点 3：搭子关系的“共识卡”机制（羁绊落地）

成为搭子后自动生成：

- **沟通公约 3 条**（对齐节奏、职责、情绪表达）  
- **共同目标 1 个**（例如“本周上 1 个小段位 / 练会 1 个英雄组合”）

### 创新点 4：论坛×推荐双通路提升匹配成功率

不同用户习惯不同：有人喜欢刷推荐，有人喜欢发帖招募。双通路可显著提升“匹配发生率”，并形成社区活跃度。

## 6. 功能规划（MVP/扩展）

> 建议先做 MVP，保证 3 人团队 3 周可完成可演示闭环。

### 6.1 MVP（必做）

#### A. 用户建档与定制

- 建档问卷（建议 12 项以内）：
  - 昵称、段位/水平（自评）、常玩时段、城市/时区  
  - **常玩游戏（可多选，含 MOBA/FPS/派对等）**、自评水平（通用档位）、主位置或分工（跨游戏选项）、风格（稳健/激进/运营/打架）  
  - 组队目标（上分/娱乐/练英雄/固定队）  
  - 沟通偏好（语音/文字、直给/委婉）  
  - 雷区标签（可选）  
  - 智能体人设（温柔队友/战术教练/搞笑搭子等）与语气强度；**可选**：专属智能体展示名、口头禅/禁忌补充（见《个人定制智能体功能设计》V1.1，可 P1 完整设计器）

#### B. 搭子名片生成与展示

- 名片卡展示：标签、宣言、组队规则、可复制/可用于发帖引用

#### C. 论坛 / 搭子广场（**核心功能**，与推荐并列）

- **列表**：分区筛选（招募/攻略/闲聊/赛事）、关键词搜索、置顶与回复数展示  
- **详情**：正文与标签、评论区占位（P1 接评论 API）、申请搭子入口  
- **发布**：选分区、快捷标签、**AI 一键生成招募草稿**、发布进列表（当前内存仓库，可换 Retrofit）  
- 招募帖可与画像字段结构化对齐（时段、目标、语音偏好等，见《论坛模块功能设计》）

#### D. AI 一键发帖（论坛增强）

- 基于画像自动生成招募帖正文（更容易被回应）  
- 发帖前文明提示（避免攻击性措辞）

#### E. 匹配推荐流

- 推荐 Top N 搭子  
- 展示：匹配度、推荐理由、冲突提示与建议

#### F. 搭子关系管理

- 申请搭子、通过/拒绝、解除搭子  
- 成为搭子时生成共识卡（沟通公约+共同目标）

### 6.2 扩展（加分，可选）

- 私信聊天（简版会话即可）  
- 标签筛选/收藏/举报/屏蔽  
- 轻复盘：每次组队后输出“下次改进 1 点”并沉淀到成长页  
- 同城活动板块（观赛、线下开黑、社团活动组织）

## 7. 关键页面与交互

### 7.1 页面清单

- `Onboarding`：建档问卷 + 人设选择  
- `BuddyCard`：名片展示/编辑  
- `Feed`：推荐搭子流（含推荐理由）  
- `Forum`：板块/帖子列表  
- `PostEditor`：发帖（AI 生成按钮）  
- `PostDetail`：回帖 + 申请入口  
- `Requests`：申请管理  
- `BuddyRoom`：搭子关系页（共识卡/共同目标）

### 7.2 关键交互设计

- **低门槛建档**：10–12 问快速完成，生成名片立即反馈  
- **卡片化输出**：名片、推荐理由、共识卡都用卡片呈现，便于演示与理解  
- **可解释推荐**：每个推荐明确“为什么”与“怎么磨合”

## 8. 技术方案（架构/数据/接口）

### 8.1 总体架构

- **前端**：Web/H5 —— **首选 Vue 3 + Vite + Element Plus**（组合式 API + Pinia + Axios + Vue Router），实现建档、论坛、推荐流、关系页；详见《Web前端技术栈选型-Vue3》.md（备选：React + Ant Design 等）  
- **后端**：REST API（Node/Nest 或 Python/FastAPI）  
- **数据库**：SQLite（开发快）或 Postgres（更规范）  
- **AI 能力**：画像总结、名片生成、发帖生成、推荐解释、共识卡生成、文明提示

### 8.2 数据模型（最小可用）

- `users`：用户基础信息  
- `profiles`：用户画像（时间、目标、位置、沟通偏好、风格等）  
- `posts`：论坛帖子（含结构化标签字段）  
- `comments`：回帖  
- `buddy_requests`：搭子申请记录  
- `buddies`：搭子关系  
-（可选）`messages`：私信消息

### 8.3 匹配评分（可解释）

建议按权重计算匹配度 \(Score\)（示例）：

- 时间重合 30%  
- 目标一致 25%  
- 位置互补 20%  
- 沟通偏好 15%  
- 风格相容 10%  
- **一票否决**：雷区冲突（如“必须语音”vs“拒绝语音”）

输出结构：`score + reasons[3] + risk[1] + advice[1]`

### 8.4 API 清单（示例）

- `POST /api/profile`：创建/更新画像  
- `GET /api/profile/me`：获取我的画像与名片  
- `POST /api/ai/buddy-card`：生成/刷新名片  
- `GET /api/recommendations`：获取推荐搭子列表  
- `POST /api/posts`：发帖（支持 AI 生成正文）  
- `GET /api/posts`：帖子列表  
- `GET /api/posts/:id`：帖子详情  
- `POST /api/posts/:id/comments`：回帖  
- `POST /api/buddy-requests`：发起搭子申请  
- `PATCH /api/buddy-requests/:id`：同意/拒绝  
- `GET /api/buddies`：我的搭子列表  
- `POST /api/ai/consensus-card`：生成共识卡

## 9. 合规与安全策略

- **不冒充真人选手/教练身份**：智能体为“虚拟搭子”，不暗示真实职业背书  
- **素材合规**：未授权前不对外传播官方素材；对外演示使用抽象化或自制素材  
- **文明社区**：发布与回帖进行文明提示；提供举报/屏蔽（MVP 可做简版）  
- **隐私与可控**：用户可随时清空画像/删除帖子/解除搭子关系（至少提供清空画像）

## 10. 指标与验收标准

- **效率指标**：建档完成后 30 秒内可获得名片与推荐列表  
- **转化指标**：推荐页 → 发起申请率；论坛招募帖 → 被回应率  
- **关系指标**：申请通过率、搭子关系 7 日留存率（可用演示数据）  
- **体验指标**：对“推荐解释”满意度（简易 1–5 评分）

## 11. 开发计划（3 周排期）

## 12. 演示脚本（2–3 分钟）

1) **建档 10 问**（30 秒）→ 生成“搭子名片”  
2) 点击 **AI 一键生成招募帖** → 发布到论坛  
3) 打开 **推荐流**：展示一个高匹配对象的“推荐理由/冲突提示”  
4) 发起申请 → 通过 → 成为搭子  
5) 自动生成 **共识卡**（沟通公约+共同目标）并展示
