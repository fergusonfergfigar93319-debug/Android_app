# TX_ku（同频搭）Android 客户端 — 前端做了什么 & 流程说明

**读者**：后端、产品、测试（需要知道「App 里已经能跑通什么、用户怎么点、数据在哪」）。  
**说明**：本文档中的「前端」指 **本仓库 Android 应用（`app` 模块）**，不包含 Web/H5。  
**关联文档**：接口契约与字段映射见 **`TX_ku_Android前后端对接说明.md`**。

## 一、总览：前端当前在干什么

| 维度 | 现状 |
|------|------|
| 网络请求 | **未接真实后端**；无 Retrofit 调用链，`ApiConstants.BASE_URL` 仅为占位。 |
| 登录注册 | **内存假账号表**（`AuthRepository`），进程杀死后除已写 SharedPreferences 外账号会丢。 |
| 用户画像 | 建档结果进 **内存 `CurrentUser.profile`**，无服务端同步。 |
| 论坛 | **内存种子帖 + 本地新发帖**；点赞/收藏仅本机；审核态在模型里已预留。 |
| 发现页 | 资讯与推荐搭子均为 **本地 mock 延迟加载**。 |
| 关注/私信 | **内存**；用户 ID 解析靠 **演示字典**（`UserDirectory`）。 |
| 智能体 | 形象/语气/备忘等 **SharedPreferences 按邮箱持久化**；回复为 **本地规则 + 关键词路由**，非大模型 API。 |

结论：**交互与界面已搭完整条产品路径**；与账号、内容、关系、AI 相关的 **权威数据应在后端落地**，前端后续改为请求接口即可。

## 二、已实现功能清单（按模块）

### 2.1 启动与账号

| 功能 | 前端行为 | 数据落点 |
|------|----------|----------|
| 启动页 | 约 0.9s 后根据状态跳转 | — |
| 注册 | 校验邮箱/密码/昵称；写入假账号表 | 内存 Map |
| 登录 | 校验密码；写入 `CurrentUser.account` | 内存 |
| 退出 | 清空会话、关注、私信线程 | `CurrentUser.clearSession()` |

### 2.2 建档（Onboarding）

| 功能 | 前端行为 | 数据落点 |
|------|----------|----------|
| 问卷填写 | 多题单选/多选/文本；必答校验 | `OnboardingViewModel` 状态 |
| 提交建档 | 合成 `Profile`（含 `userId` 本地时间戳）、mock 生成 `BuddyCard` | `CurrentUser.profile`、`buddyCard` |
| 与注册信息合并 | 昵称、头像与账号侧对齐 | 同步 `AuthRepository.updateStoredProfile` |
| 智能体存档刷新 | 调 `UserAgentStore.load/save` | SharedPreferences |

### 2.3 游戏兴趣（Follow Games）

| 功能 | 前端行为 | 数据落点 |
|------|----------|----------|
| 多选常玩游戏 | 与资讯频道等对齐 | `GameInterestStore`（按邮箱） |
| 完成标记 | 未完成则每次启动挡在主 Tab 前 | `hasCompletedSelection()` |

### 2.4 主界面：底部四 Tab

| Tab | 名称 | 前端内容 |
|-----|------|----------|
| 发现 | 资讯流 + 子 Tab「推荐搭子」 | 假数据列表、下拉刷新（仍假）、跳转广场/搭子等 |
| 搭子 | 智能体创作台 | 成品卡、气质套组、备忘/忌讳/手写总则、Chip 细调、解锁聊天、进聊天 |
| 广场 | 论坛 | 分区、列表、搜索、进详情、FAB 发帖 |
| 我的 | 个人中心 | 资料编辑入口、关注列表、按 ID 加友、设置与退出等 |

**悬浮入口**：已建档且已解锁聊天时，在非「搭子」Tab 显示 **进智能体聊天** 的悬浮气泡（含模拟未读提醒文案）。

### 2.5 论坛

| 功能 | 前端行为 |
|------|----------|
| 列表 | 读 `ForumRepository.posts`；过滤未过审帖 |
| 分区/搜索 | 本地过滤 |
| 发帖 | 标题正文标签、本地相册/视频 URI；`prepend` 到列表 |
| 详情 | 评论列表 + 发评；`replyCount` 本地递增 |
| 点赞/收藏 | 仅更新本地 StateFlow 与帖上数字 |

### 2.6 智能体（搭子）

| 功能 | 前端行为 |
|------|----------|
| 形象与展示名 | 多套 drawable、可改名、可选边框/气泡等 |
| 成品搭子一键套用 | 写入完整 `AgentTuning` |
| 聊天 | 需先「解锁」；气泡样式跟 tuning；快捷短语 |
| 回复逻辑 | `AgentPersonaResolver` + `AgentTaskRouter`（含跳转发帖/广场等指令） |
| 活动提醒卡片 | ViewModel 内定时注入，模拟推送 |
| 聊天主题色 | `AgentChatPrefsStore` 按账号持久化 |

### 2.7 社交

| 功能 | 前端行为 |
|------|----------|
| 按用户 ID 查找 | `UserDirectory` 演示数据 |
| 关注/取关 | `FollowRepository`；部分 ID 演示「自动回关」 |
| 私信 | 须先关注；未互关最多发 1 条给对方；互关不限 |

### 2.8 其他全屏页

- 帖子详情、发帖编辑器  
- 资料编辑（`ProfileEditScreen`）  
- 搭子房（`BuddyRoomScreen`，按 `relationId`）  
- 与上述路由在 `BuddyCardNavHost` 中注册

## 三、用户侧主流程（端到端）

### 3.1 新用户首次使用

```text
打开 App
  → 启动页
  → 注册 / 登录（本地假账号）
  → 建档问卷（必填项完成才可提交）
  → 游戏兴趣多选（未完成则进不了主 Tab）
  → 进入主界面（默认「发现」Tab）
```

后续用户可自由切换：发现 / 搭子 / 广场 / 我的；从发现页头可通过桥接跳到指定广场筛选或搭子 Tab。

### 3.2 老用户再次打开（同进程内已登录）

```text
启动页
  → 从 SharedPreferences 恢复智能体偏好（UserAgentStore）
  → 若 profile 为空 → 仍去建档
  → 若未完成游戏兴趣 → 游戏兴趣页
  → 否则直接进入主 Tab
```

**注意**：当前「登录态」主要在内存；**杀进程后** 若未做持久化登录，会回到登录页（与后续接 Token 持久化后的行为会不同）。

### 3.3 智能体：从捏人到聊天

```text
底栏「搭子」
  →（可选）点成品卡 / 气质套组 / 展开 Chip 细调
  → 填写备忘、忌讳、手写总则、快捷句等
  → 点「完成创作，解锁聊天」（写 agentChatUnlocked = true）
  → 可「开聊」进入会话页，或之后在其它 Tab 点悬浮球进入
```

未解锁时直接进入聊天路由会弹窗引导回搭子页。

### 3.4 论坛：看帖与互动

```text
底栏「广场」
  → 选分区 / 搜索
  → 点帖进详情，看评论、发评论
  → 点赞、收藏（当前仅本地）
  → FAB 打开发帖页，发帖后回到列表可见新帖（本地）
```

### 3.5 推荐搭子（发现 Tab 内）

```text
发现 → 切换到「推荐」子 Tab
  → 展示 mock 推荐卡片（含匹配分、理由、风险、建议、名片摘要）
  → 「发起申请」按钮已预留，内部 TODO，未发网络请求
```

### 3.6 社交：加关注与私信

```text
我的 → 按 ID 搜索用户（演示 ID 可搜到）
  → 关注
  → 若演示账号支持自动回关 → 视为互关
  → 进入私信页发消息（未互关时对方只能收到你 1 条，规则见代码）
```

## 四、前端数据流简图（给后端对齐用）

```text
                    ┌─────────────────────┐
                    │   CurrentUser       │
                    │ account / profile   │
                    │ buddyCard / buddyAgent│
                    │ agentTuning         │
                    └─────────┬───────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
 AuthRepository        ForumRepository      FollowRepository / DM
 （内存账号）            （内存帖与评论）         （内存关系与私信）
        │                     │
        │              UserAgentStore / GameInterestStore /
        │              AgentChatPrefsStore（SharedPreferences）
        ▼
   接后端后：上述模块改为 Repository → API → 服务端权威数据
```

## 五、前端明确未做 / 占位（后端可优先补）

- 无 JWT 存储与刷新、无统一 HTTP 客户端。  
- 建档、名片、推荐申请、关注、私信、帖子/评论的 **持久化与多端同步**。  
- 媒体上传：发帖仍带本地 Uri，**无对象存储上传链路**。  
- 智能体 **无服务端对话**；活动提醒为 **本地定时器**，非推送。  
- `BuddyRoom`、共识卡等 **页面有，业务数据未接网**。
