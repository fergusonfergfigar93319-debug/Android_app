# 论坛与媒体 · 内容审核（后端设计）

> **归属**：审核策略、队列、人审、媒体扫描均在 **后端** 实现；客户端仅展示 `moderation_status` 合法态下的帖子，并上传媒体至对象存储由服务端校验。  
> **客户端对齐**：见 `Post` / `PostModerationStatus`、`Post.isVisibleInPublicForum()`；种子数据见 `ForumRepository`（含 `PENDING_REVIEW` 演示帖，广场列表已过滤）。**Android 与后端接口的完整约定**（详情 404、作者可见、`GET /users/me/posts` 建议、评论拒评、本地演示自动过审等）见 **`TX_ku_Android前后端对接说明.md`** §2.7、§4.4。

---

## 1. 目标与原则

| 目标 | 说明 |
|------|------|
| 合规 | 拦截违法违规、色情暴力、诈骗引流、隐私泄露等 |
| 体验 | 先审后发或「先发后审」可配置；审核中作者可见状态 |
| 媒体 | 图片/视频除文本规则外，走 **哈希黑名单、色情暴力模型、文件头与大小限制** |
| 可审计 | 审核动作落库：操作者、时间、结果、原因码 |

**原则**：审核结论以后端为准；列表接口 **默认不返回** `REJECTED` / `PENDING_REVIEW`（公域）；作者「我的帖子」接口可返回全状态。

## 2. 数据模型（建议）

### 2.1 帖子表 `posts` 扩展字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `moderation_status` | ENUM | `APPROVED` / `PENDING_REVIEW` / `REJECTED` / `MACHINE_FLAGGED` |
| `moderation_source` | VARCHAR | `none` / `text_model` / `image_model` / `keyword` / `manual` |
| `moderation_score` | FLOAT | 机审综合分，可选 |
| `moderation_labels` | JSONB | 命中标签，如 `["abuse","spam"]` |
| `moderation_hint` | TEXT | 对用户可见的简短原因（拒审/补充材料） |
| `reviewed_at` | TIMESTAMPTZ | 终审时间 |
| `reviewer_id` | VARCHAR | 人审账号，机审为空 |

### 2.2 媒体表 `post_media`（若独立存储）

| 字段 | 说明 |
|------|------|
| `media_id`, `post_id`, `url`, `mime`, `size_bytes`, `width`, `height` | 基础 |
| `scan_status` | `pending` / `pass` / `block` |
| `scan_detail` | JSON，模型输出摘要 |

## 3. 流程（推荐）

### 3.1 发布 `POST /posts`

1. **鉴权** → 校验标题/正文长度、标签白名单、频率限制（防刷）。  
2. **文本机审**：敏感词、辱骂、联系方式滥用、LLM 分类（可选）。  
3. **媒体**：客户端先 `POST /uploads/presign` 拿直传 URL → 上传完成后 `POST /posts` 带 `media_ids[]`。服务端对每文件做 **病毒扫描 + 视觉审核 + 分辨率/时长**；未全部 `pass` 则帖子置 `MACHINE_FLAGGED` 或 `PENDING_REVIEW`。  
4. **策略分支**：  
   - **严格**：全部通过前 `moderation_status = PENDING_REVIEW`，**不进入公域列表**。  
   - **宽松**：可先 `APPROVED` 展示，异步复审，命中则降级为 `MACHINE_FLAGGED` 或下架。  
5. 返回帖子对象（含 `moderation_status`），客户端按状态展示文案。

### 3.2 列表 `GET /posts`

- Query：`status=public` 时 **仅** `APPROVED`（及业务上允许展示的 `MACHINE_FLAGGED` 降权策略若启用）。  
- 与客户端 `isVisibleInPublicForum()` 保持一致。

### 3.3 人审后台（内部）

- `GET /admin/moderation/queue?status=PENDING_REVIEW`  
- `PATCH /admin/moderation/posts/{id}` Body：`{ "action": "APPROVE" | "REJECT", "reason_code": "...", "hint": "..." }`  
- 操作写审计日志。

### 3.4 举报与复审

- `POST /posts/{id}/reports`：用户举报 → 队列优先级提升或自动 `PENDING_REVIEW`。  
- 可选：同一帖子多次举报触发隐藏（仍由后端改状态）。

## 4. API 补充（与《同频搭_API与数据定义文档》衔接，客户端产品名：**元流同频**）

### 4.1 `POST /posts` Request 扩展

```json
{
  "title": "...",
  "content": "...",
  "tags": ["..."],
  "category_id": "recruit",
  "media_ids": ["med_001", "med_002"]
}
```

### 4.2 `POST /posts` Response `data` 扩展

```json
{
  "post_id": "p_xxx",
  "moderation_status": "PENDING_REVIEW",
  "moderation_hint": "内容将在数分钟内完成审核，通过后将出现在广场。",
  "visible_in_public": false
}
```

### 4.3 错误码（建议）

| code | 含义 |
|------|------|
| 40021 | 文本命中敏感词，拒绝入库 |
| 40022 | 媒体未通过安全扫描 |
| 40023 | 发布频率过高 |
| 40310 | 用户因违规被禁言 |

## 5. 与当前 App 演示数据

- **内置图**：`res/drawable-nodpi/forum_demo_*.png`（招募/攻略/社交主题示意），种子帖 `p1`/`p3`/`p4`/`p5` 已挂载；Coil 通过 `android.resource://` 加载。  
- **审核演示**：`p_pending_demo` 为 `PENDING_REVIEW`，**不应**出现在广场列表（`ForumViewModel` 已按 `isVisibleInPublicForum()` 过滤）。  
- **用户新发帖**：客户端默认写入 `PENDING_REVIEW`；内存演示下约 **90 秒** 自动改为 `APPROVED`（仅本地、无后端时验证闭环，**上线应关闭**）。  
- **作者侧**：`ProfileScreen`「我的 · 帖子」列出本人全部审核态；`PostDetailScreen` 对非作者未过审帖不展示正文。

## 6. 运维与隐私

- 日志脱敏：正文仅保留 hash 或截断片段。  
- 审核数据保留周期符合当地法规。  
- 未成年人模式：可强制「先审后发」与更严媒体策略。
