# 《个人定制智能体功能设计》

> 与《同频搭_选题设计方案》并列：**智能体为产品核心能力之一**，与建档画像、推荐流、论坛发帖形成闭环。  
> **V1.1 重点**：用户主导设计专属智能体 + 多槽位与 System 片段（规划）。  
> **V1.2 客户端已落地**：**形象渐变卡**、**6 组表达维度 Chip**（含情绪底色 / 玩梗浓度 / 称呼习惯）、**4 组快捷预设**、**对话气泡预览**；与 `AgentPersonaResolver` / 发帖文风条联动。

---

## 1. 定位与设计理念

### 1.1 双层结构（避免与建档混淆）

| 层级 | 内容 | 谁主导 | 变更频率 |
|------|------|--------|----------|
| **长期画像** | 游戏、时段、位置、目标、沟通偏好、雷区等 | 建档问卷 / 资料编辑 | 低 |
| **专属智能体配置** | 展示名、称呼方式、角色皮、音色语义、语气模板、禁忌词、口头禅、场景微调 | **用户在设计器中自选 + 系统补全** | 中高 |

- **建档**：决定「是谁、玩什么、什么风格」——用于 **匹配与名片事实字段**。  
- **定制智能体**：在画像之上，提供 **可设计的虚拟搭子人格**——用于 **文案生成、解释话术、模拟对话、界面皮肤**，并可通过 **多套预设（槽位）** 切换「上分严肃体 / 娱乐整活体」等。  
- **价值**：降低「写招募 / 开口沟通」成本；**情感归属**（「这是我的 Agent」）提升留存；与发帖、推荐解释共用同一套 **可序列化的 Agent Config**。

### 1.2 设计原则

1. **用户可感知专属**：必须有 **自定义展示名**（或从昵称衍生 + 可改）、**一句对我怎么称呼**、可选 **口头禅/结束语**。  
2. **低门槛 + 可深度**：默认 **模板一键套用**；进阶用户可展开 **高级：禁忌词、额外指令（短文本，有长度与审核上限）**。  
3. **安全与合规**：智能体仍为「虚拟搭子」，**不冒充真人职业选手/教练**；用户自定义文案走 **本地敏感提示 + 服务端审核（若有）**；禁止收集过度隐私作为「人设」卖点。  
4. **与 AI 对齐**：任何界面上的选项都能映射为 **结构化 JSON + 一段固定模板的 System 片段**，便于后端/边缘模型消费。

## 2. 用户动线：「智能体设计器」

### 2.1 主流程（建议 UI 步骤）

1. **入口（点击创作）**：底栏 Tab「**创作**」→ 进入 **创作专属智能体** 页；首屏为 **主题渐变形象卡**（随 `uiThemeKey` 变色）+ **快捷形象预设** + **分组定制卡片**（每项带一句说明）+ **对话气泡式模拟预览**。推荐流 Banner、我的页卡片文案统一为 **「点击创作专属智能体」**。  
2. **步骤 A · 命名与关系**  
   - **智能体展示名**（2–12 字，默认由系统根据画像生成，可改）。  
   - **对我的称呼**（如「队长」「兄弟」「搭档」，可选）。  
3. **步骤 B · 人格基底**（与建档 `personalityArchetype` 联动，可覆盖为「同档另一模板」）  
   - 预设：**温柔队友 / 战术教练 / 搞笑搭子 / 冷静分析 / 热血鼓励**（与问卷选项对齐）。  
4. **步骤 C · 外观与音色语义**（与 `agentVisualTheme`、`agentVoicePref` 对齐，可在此页二次确认）  
   - **角色皮**：分工映射默认皮 + 用户从 **皮肤库** 中选替代（仅语义与 UI key，不涉及侵权素材）。  
   - **音色语义**：语速/口癖标签（与 TTS 或纯文案风格一致）。  
5. **步骤 D · 实时表达参数**（`AgentTuning`，Android 已实现）  
   - 基础：`intensity`、`replyLength`、`focusScenario`。  
   - 扩展：`emotionTone`（共情/理性/热血）、`humorMix`（严肃/适中/玩梗）、`addressStyle`（昵称/中性/尊称）。  
   - **快捷预设**：冲分招募、娱乐开黑、降压复盘、战术教练（一键写回多字段）。  
6. **步骤 E · 我的专属句**（新增，核心个性化）  
   - **口头禅**（可选，≤20 字）  
   - **结束语**（可选，≤30 字）  
   - **不要这样说**（禁忌/雷区表述，≤100 字，与画像 `noGos` 合并参与 Prompt）  
   - **补充人设**（可选，≤200 字，**引导用户写玩法与沟通偏好，禁止辱骂与歧视**）  
7. **预览**：**模拟对话 2 轮**（已有能力扩展读取上述字段）→ 不满意返回修改。  
8. **保存**：写入 **当前方案**；可选 **另存为新方案**（多槽位，见 3.2）。

### 2.2 与现有入口的关系

- **推荐流 Banner / 我的页**：主文案 **「点击创作专属智能体」**，点击进入与底栏「创作」Tab 同一创作页。  
- **发帖页文风条**：展示 **当前方案** 摘要（展示名 + 语气 + 场景）。  
- **无画像**：创作页内引导 **「去建档，解锁创作」**；建档完成后再进入同一页完成创作闭环。

## 3. 数据模型

### 3.1 客户端扩展示意（与现有 Kotlin 对齐方向）

在保留 `Profile`、`AgentTuning`、`BuddyAgentPersona` 的前提下，建议新增 **用户可编辑配置层**（命名可二选一）：

**方案 A：扩展 `AgentTuning` → 改名为 `AgentConfig`（推荐后续重构）**

| 字段 | 说明 |
|------|------|
| 继承现有 | `intensity`、`replyLength`、`focusScenario` |
| `customDisplayName` | 用户自定义展示名，空则使用 `BuddyAgentPersona.displayName` |
| `addressingMe` | 对我的称呼，可空 |
| `catchphrase` | 口头禅，可空 |
| `signOff` | 结束语，可空 |
| `avoidPhrases` | 不要这样说的自由文本，可空 |
| `extraPersonaNotes` | 补充人设，可空（长度限制 + 合规） |
| `personaPresetId` | 选的基底模板 ID（与问卷 archetype 可不一致时记录覆盖） |

**`BuddyAgentPersona`**：仍由 `AgentPersonaResolver.resolve(profile, config)` 合成；**合成规则**为：`customDisplayName` 优先于默认合成名，其余字段进入 **traits / sampleDialogue / polish 逻辑**。

**方案 B：独立 `BuddyAgentUserDesign` + `AgentTuning`**（渐进式）

- 不破坏现有 `AgentTuning` 三字段 API；用户专属句放在 `BuddyAgentUserDesign`，合并时再传给 Resolver。

### 3.2 多方案（槽位）

- 用户可保存 **最多 N 套**（建议 MVP：2–3 套），例如：「冲分专用」「娱乐嘴替」「复盘模式」。  
- **当前启用**：`activeSlotId`；切换后全局 AI 调用使用该槽配置。  
- 与 **名片**：名片仍反映 **真实画像**；智能体仅为 **交互与生成层**，避免「名片人设」与「智能体皮」完全割裂——在 UI 文案上说明：**「搭子名片是真实组队信息，智能体是帮你说话的风格」**。

### 3.3 下发给 AI 的 System 片段（逻辑结构）

生成招募帖、推荐解释、共识卡草案时，后端或客户端组装 **不可见** 的结构化指令，例如：

- 角色：虚拟同频搭智能体助手，非真人教练。  
- 用户画像摘要：（来自 `profile`）  
- 智能体人格：（`preset + customDisplayName + addressingMe + catchphrase + signOff`）  
- 表达：`intensity` / `replyLength` / `focusScenario`  
- 禁止：`noGos` ∪ `avoidPhrases` 解析后的列表  
- 补充：`extraPersonaNotes`（过滤后）

**输出要求**：与现有接口返回字段一致（标题、正文、reasons 等），仅在风格上体现用户设计。

## 4. 业务规则

- **首次/重新建档完成**：`agentTuning`（或 `AgentConfig`）中非「用户显式保存过」的字段可重置默认；若用户曾 **命名/槽位** 且选择「保留我的智能体设计」，则只刷新与画像强相关的 **默认合成部分**（如角色皮映射），不覆盖 `customDisplayName`。  
- **在设计器内修改**：立即 `recomputePersona()`，并刷新 **我的页预览、发帖页文风条、Banner**。  
- **恢复默认**：仅重置 **实时微调三件套 + 专属句**；可选「全部恢复」连展示名一并重置（需二次确认）。  
- **合规**：发布类 AI 输出仍走 **文明提示**；用户 `extraPersonaNotes` 若含辱骂等，客户端预检拦截或打回修改。

## 6. 接口与持久化（建议）

详见《同频搭_API与数据定义文档》**智能体配置**小节。要点：

- `GET /profiles/me` 的 `data` 中增加 `agent_config`（或独立 `GET /agent-config`）。  
- `PUT /agent-config`：保存当前槽位与 `active_slot_id`。  
- 与 `POST /ai/posts`、`POST /ai/buddy-card` 请求体可选携带 `agent_config_snapshot`（或由服务端按 user 拉取）。

## 7. 验收与演示要点

- 用户能在 **30 秒内** 完成：改名 + 选基底 + 调 3 个 Chip + 保存。  
- 演示话术：**「这是我的专属智能体，发帖和解释都会用它的语气」**。  
- 切换槽位后，**模拟对话与 AI 草稿** 风格可见差异。
