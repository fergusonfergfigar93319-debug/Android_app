## **1\. 前端架构与开发规范 (Architecture & Standards)**

### **1.1 核心架构：MVI (Model-View-Intent)**

全端采用严格的单向数据流（Unidirectional Data Flow）。

* **Model (ViewState)**：页面唯一的数据源，使用 data class 密封类表示界面的各种状态（加载中、内容展示、错误提示）。  
* **View (Compose)**：纯 UI 层，只负责监听 State 渲染界面，并将用户操作转发为 Intent。  
* **Intent (Action)**：用户意图或系统事件的封装。  
* **ViewModel**：接收 Intent，调用领域层/数据层逻辑，更新 State。

### **1.2 状态定义基类规范**

所有 ViewModel 需遵循以下状态封装模式：

sealed class UiState\<out T\> {  
    object Loading : UiState\<Nothing\>()  
    data class Success\<T\>(val data: T) : UiState\<T\>()  
    data class Error(val message: String) : UiState\<Nothing\>()  
}

// 示例 Intent  
sealed class FeedIntent {  
    object Refresh : FeedIntent()  
    data class SendRequest(val targetUserId: String) : FeedIntent()  
}

## **3\. 核心模块开发指南 (Core Modules Implementation)**

### **3.1 用户建档模块 (Onboarding)**

* **需求**：10-12 道题目的多步表单，完成后调用 AI 接口生成名片。  
* **UI 策略**：使用 Compose 的 HorizontalPager 实现平滑的卡片式滑动答题。  
* **状态管理**：ViewModel 中维护一个 Map\<QuestionId, Answer\>，当答题进度到 100% 时触发 SubmitIntent。

### **3.2 推荐搭子流模块 (Feed)**

* **需求**：展示 Top N 匹配用户，突出展示“推荐理由”和“潜在冲突”。  
* **UI 策略**：使用 LazyColumn。卡片设计采用多层级阴影与圆角（高级感）。  
* **关键代码实现 (MVI \+ Compose 示例)**：

@Composable  
fun FeedScreen(viewModel: FeedViewModel \= hiltViewModel()) {  
    val uiState by viewModel.uiState.collectAsState()

    when (val state \= uiState) {  
        is UiState.Loading \-\> BuddyCardSkeletonList() // 骨架屏动画  
        is UiState.Error \-\> ErrorScreen(state.message) { viewModel.handleIntent(FeedIntent.Refresh) }  
        is UiState.Success \-\> {  
            LazyColumn(  
                contentPadding \= PaddingValues(16.dp),  
                verticalArrangement \= Arrangement.spacedBy(16.dp)  
            ) {  
                items(state.data) { recommendation \-\>  
                    RecommendationCard(  
                        data \= recommendation,  
                        onRequestClick \= { viewModel.handleIntent(FeedIntent.SendRequest(it)) }  
                    )  
                }  
            }  
        }  
    }  
}

### **3.3 论坛与 AI 一键发帖模块 (Forum)**

* **需求**：帖子列表、详情、AI 辅助写帖。  
* **痛点**：AI 生成正文耗时，需要防阻断设计。  
* **UI 策略**：点击“AI 一键生成”后，弹出 BottomSheet 或在文本框内显示 Shimmer (微光动画) 骨架屏，生成完毕后文字通过打字机效果 (Typewriter Effect) 逐字渲染，提升产品调性。

### **3.4 搭子关系与共识卡模块 (BuddyRoom)**

* **需求**：展示达成关系后的“沟通公约”与“共同目标”。  
* **UI 策略**：此页面作为应用的“奖励页”，UI 需极具仪式感。顶部展示双方头像（结合特效连线），下方以“契约书”形式展示共识卡。  
* **数据绑定**：监听本地 Room 数据库的 Flow，实现关系状态的实时 UI 更新。

## **4\. 路由与导航设计 (Navigation)**

使用 Jetpack Navigation Compose 进行全应用路由管理。

@Composable  
fun BuddyCardNavHost(navController: NavHostController) {  
    NavHost(navController \= navController, startDestination \= "splash") {  
        // 启动页/逻辑分发  
        composable("splash") { SplashScreen(navController) }  
          
        // 建档流程  
        composable("onboarding") { OnboardingScreen(navController) }  
          
        // 主界面 (底部导航栏承载)  
        composable("main\_tabs") { MainTabScreen(navController) }  
          
        // 论坛帖子详情 (传参)  
        composable(  
            route \= "post\_detail/{postId}",  
            arguments \= listOf(navArgument("postId") { type \= NavType.StringType })  
        ) { backStackEntry \-\>  
            val postId \= backStackEntry.arguments?.getString("postId")  
            PostDetailScreen(postId, navController)  
        }  
          
        // 专属搭子关系房  
        composable("buddy\_room/{relationId}") { ... }  
    }  
}

## **5\. 公共 UI 与设计系统 (Design System)**

为了保证快速迭代，团队必须严格使用 core/designsystem 中的组件，禁止在业务代码中硬编码颜色和字体大小。

### **5.1 颜色调色板 (Colors)**

* **Primary**：电竞紫 / 赛博蓝 (品牌色，用于主要按钮、进度条)  
* **Secondary**：柔和薄荷绿 (用于高亮“匹配率”、“推荐理由”)  
* **Warning**：珊瑚红 (用于标注“雷区”、“潜在冲突”)  
* **Background**：深色模式为主 (Dark Theme)，背景采用 MaterialTheme.colors.background (\#121212 搭配局部渐变)。

### **5.2 核心复用组件 (Reusable Composables)**

**组件 1：标签 (TagChips)**

大量用于展示风格、位置。

@Composable  
fun BuddyTag(text: String, isHighlight: Boolean \= false) {  
    Surface(  
        shape \= RoundedCornerShape(16.dp),  
        color \= if (isHighlight) MaterialTheme.colorScheme.primary.copy(alpha \= 0.1f)   
                else MaterialTheme.colorScheme.surfaceVariant

        Text(  
            text \= text,  
            modifier \= Modifier.padding(horizontal \= 12.dp, vertical \= 6.dp),  
            style \= MaterialTheme.typography.labelMedium,  
            color \= if (isHighlight) MaterialTheme.colorScheme.primary   
                    else MaterialTheme.colorScheme.onSurfaceVariant  
        )  
    }  
}

## **6\. 性能与体验优化规范 (Performance & UX)**

1. **避免无意义的重组 (Recomposition)**：在 LazyColumn 中，必须为 items 提供稳定的 key，例如 key \= { it.userId }。  
2. **状态防抖 (Debounce)**：对按钮点击和列表点赞等高频操作，ViewModel 层需引入防抖处理（Coroutine delay 配合时间戳判断）。  
3. **沉浸式体验**：集成 Accompanist System UI Controller，实现状态栏与应用顶栏的沉浸式融合，符合电竞类应用的高级调性。  
4. **AI 等待焦虑缓解**：禁止使用单纯的转圈 CircularProgressIndicator 来等待 AI。必须使用占位符微光动画 (Shimmer) 配以文案（例如：“正在分析您的对局习惯…” \-\> “正在匹配灵魂契合度…” 动态切换）。
