# 《论坛模块功能设计》— 核心功能之一

**版本**：V1.2  
**端侧**：Android（Jetpack Compose）为主，接口与数据结构可对齐 Web  
**定位**：与「推荐流」并列的 **双通路找搭子** —— 用户可发帖招募、浏览帖子、进入详情发起「申请搭子」，形成社区活跃与转化闭环。

---

## 1. 产品定位与目标

| 维度 | 说明 |
|------|------|
| **核心价值** | 降低「写招募帖」成本（AI 辅助），提高曝光与回应率；补充算法推荐覆盖不到的长尾需求。 |
| **与首页（Feed）关系** | **首页** = 资讯 / 官方 / 找搭子推荐（`FeedScreen` + 同频搭顶栏）；**广场** = 用户帖子流（`ForumScreen`）。二者 **入口、顶栏文案、背景色板均分列设计**，不在论坛顶重复「我的智能体」横幅（智能体走独立 Tab + 全局悬浮）。 |
| **与推荐流关系** | 推荐 = 系统主动推人；论坛 = 用户主动发声。二者共用画像与名片，在帖子详情/发帖流程可引用「我的名片摘要」。 |
| **成功指标（演示/首期）** | 列表可刷、**下拉刷新**、**分页加载**、可筛选、可搜索；发帖可进列表；详情 **评论列表 + 发评**；可返回并申请搭子。 |

---

## 2. 信息架构

```
论坛 Tab（广场）
├── 顶栏：标题 + 副标题（找队友 / 攻略与讨论）
├── 搜索：标题/正文/标签 本地过滤（后续接服务端搜索）
├── 分类 Chip：全部 / 招募组队 / 攻略心得 / 闲聊交友 / 赛事活动
├── 帖子列表（**PullToRefresh**；**滚动到底加载更多**；卡片含分区、回复数等）
├── FAB：发帖
├── 发帖页
│   ├── 选分区 + 标题 + 正文
│   ├── 快捷标签（可选）
│   └── AI 一键生成（模拟延迟 → 填入示例文案，可再接真实 API）
└── 帖子详情
    ├── 分区 + 元信息 + 正文
    ├── **评论列表**（`PostComment` + `commentsByPost`）
    ├── **写评论** → `ForumRepository.addComment`（同步 `replyCount`）
    └── CTA：申请搭子
```

## 3. 数据模型（客户端 + 建议后端）

### 3.1 帖子 `Post`（扩展字段）

| 字段 | 类型 | 说明 |
|------|------|------|
| postId | String | 主键 |
| categoryId | String | 分区：`recruit` / `guide` / `social` / `event` |
| authorId / authorName | String | 作者 |
| title / content | String | 标题与正文 |
| tags | List\<String\> | 展示标签 |
| createdAt | String | 展示用时间 |
| replyCount | Int | 回复数（占位 0 亦可） |
| pinned | Boolean | 置顶（可选，默认 false） |

### 3.3 评论 `PostComment`

| 字段 | 说明 |
|------|------|
| commentId / postId | 主键与所属帖 |
| authorId / authorName / content / createdAt | 展示与排序 |

客户端：`ForumRepository.commentsByPost`（`Map<postId, List<PostComment>>`），`addComment` 会追加评论并 **自增对应 Post.replyCount**。

### 3.2 分区枚举（与 UI Chip 一致）

- `all`：逻辑「全部」，不落库  
- `recruit`：招募组队（**默认发帖分区**）  
- `guide`：攻略心得  
- `social`：闲聊交友  
- `event`：赛事活动

## 4. API 约定（与《API 与数据定义文档》对齐）

- `GET /posts?category=&q=&page=`：列表 + 筛选 + 分页  
- `GET /posts/{id}`：详情（含 `replyCount`、可选 `comments` 预览）  
- `POST /posts`：发帖（body 含 `categoryId`、`title`、`content`、`tags`）  
- `POST /ai/post-draft`：根据画像生成招募帖草稿（与发帖页「AI 一键生成」对应）  
- `GET /posts/{id}/comments?page=`：评论分页  
- `POST /posts/{id}/comments`：发表评论  

---

## 4.5 列表与信息流体验（设计参考）

结合当前常见 **游戏社区 / 论坛类 App** 的做法：

- **扫读与直观**：列表中 **直接展示** 标题摘要、**首张配图 / 视频预览**，避免「仅文字提示含图」带来的不直观。  
- **快捷互动**：点赞 / 评论 / 收藏 / 分享在列表中 **常驻**。  
- **筛选与标签**：分区 Chip、热门话题、搜索组合。  
- **可读性**：正文字号与行距与 Material 体系一致；完整内容与嵌套回复在详情页。

## 5. Android 实现映射

| 能力 | 路径/说明 |
|------|-----------|
| 分区与常量 | `feature/forum/ForumCategories.kt` |
| 列表 + 评论数据源 | `ForumRepository`（`posts`、`commentsByPost`、`addComment`） |
| 列表状态 | `ForumViewModel`：`ForumListUi`（首屏、**refresh**、**loadMore**、筛选/搜索） |
| 列表 UI | `PostListScreen.kt`：**Material3 PullToRefreshBox**、**LazyColumn 触底加载**、页脚 loading/「已加载全部」；帖子卡片 **直接展示摘要 + 媒体预览** |
| 容器与 FAB | `feature/forum/ForumScreen.kt` |
| 发帖 | `feature/forum/PostEditorScreen.kt` |
| 详情 | `PostDetailScreen.kt`：**LazyColumn** 展示帖 + **评论列表** + 输入框发评 |
| 导航 | `Routes.POST_EDITOR`、`postDetail(id)` |

**状态模式**：`ForumListUi`（`isInitialLoading` / `isRefreshing` / `isLoadingMore` / `hasMore` / `errorMessage`）；分页为对 **当前筛选结果全量列表** 的内存切片（`pageSize=8`），接 API 后改为 `page` 请求。
