# 元流同频 — 智能体聊天（Android）实现汇总

**目的**：记录 **智能体会话页** 已落地的交互与数据模型，供联调、测试与 **Web 端 `/app/agent/chat` 体验对齐** 时查阅。  
**维护**：Android 侧有重大变更时更新本文；字段与枚举以仓库源码为准。

---

## P0 — 五种差异化提醒卡片

| 项 | 说明 |
|----|------|
| **数据模型** | `AgentChatStreamItem.kt` 新增 `ReminderCardType` 枚举；`EventReminder` 增加 **`cardType`** 字段，用于区分卡片语义与 UI。 |
| **Mock 演示** | `AgentChatViewModel.kt`：模拟推送由 **2 条扩展为 5 条**，**每种类型各一条**；**间隔约 40 秒**依次注入，便于 Demo **全程覆盖**五种样式。 |
| **UI** | `AgentChatScreen.kt` → `EventReminderCard`：**按 `cardType` 渲染** — 独立 **主题色**（金 / 紫 / 蓝 / 红 / 橙）、**类型角标**、**差异化操作按钮组**。 |

**Web 对齐建议**：流式消息列表若展示「活动/系统提醒卡片」，应对照 **`cardType` 语义** 做等价分区样式与操作（可用 `el-alert` / 自定义卡片 + CSS 变量映射五色）。

## P1 — 赛前战术板（Battle Prep）

| 项 | 说明 |
|----|------|
| **状态** | `AgentChatUi` 增加 **`battlePrepStep`**、**`battlePrepMode`**（状态机字段，驱动多步对话）。 |
| **触发** | `sendMessageContent` **入口优先**匹配触发词（示例：**「开黑」「打排位」** 等；完整词表以代码为准），命中后 **不走普通单条回复**，而进入 **三步流程**。 |
| **流程** | **① 选模式** → **② 选段位** → **③ 输出针对性建议**；**每一步助手回复均走流式「打字机」输出**（与常规流式消息一致）。 |

**Web 对齐建议**：Pinia 或组件内维护等价的 `battlePrepStep` / `battlePrepMode`；发送前做关键词拦截；多步中间态在消息列表中可用「系统步骤气泡」或同会话内连续流式块区分。

---

## P1 — 连败保护（Tilt Protection）

| 项 | 说明 |
|----|------|
| **检测** | **`isTiltMessage`**：用户输入命中 **10 个关键词**（具体词表以 `AgentChatViewModel` 或独立工具函数为准）时进入保护分支。 |
| **处理** | 调用 **`handleTiltProtection`**：按情绪分为约 **三类** — **队友问题 / 被压制 / 连败**，分别给出 **不同共情回复**；回复 **末尾带操作按钮**（如继续聊、去广场、调整预期等，以 UI 实现为准）。 |

**Web 对齐建议**：关键词与三类分支文案与 Android **同源文档或共享配置**（如后续抽到 `shared/tilt-keywords.json`），避免双端话术漂移。

## P1 — 英雄头像 FAB + 呼吸动画

| 项 | 说明 |
|----|------|
| **文件** | `AgentChatFloatingEntry.kt` **完全重写**。 |
| **FAB** | 由通用 FAB 改为 **圆形英雄头像**，资源 **跟随用户人设 / 智能体配置**（与搭子形象一致）。 |
| **未读态** | 有未读时：**金色边框** + **约 1.08 倍脉冲「呼吸」动画**（循环强调）。 |
| **气泡预览** | 增加 **搭子小头像**；内容区 **滑入动画**（入场动效）。 |
| **拖动与收起** | **按住圆形头像** 可拖动到屏幕其他位置（偏移 **rememberSaveable** 持久化）；**拖动过程中** 底部上方出现提示 **「拖到底部以隐藏图标」**（接近隐藏阈值时渐变为 **「松开即可隐藏」**，描边/阴影随 **nearBottomProgress** 增强）；**底部渐变 + 头像外圈光晕 + 略放大** 提供接近底部的动效反馈；**向下拖过阈值** 进入 **隐藏态**，主 FAB 收起为 **底部居中条**，**点击条** 恢复并复位偏移；气泡仍支持 **点击进入会话**。 |

**Web 对齐建议**：悬浮入口使用 **用户当前绑定英雄头像**；未读时用 ring + CSS `@keyframes` 做缩放呼吸；预览条内嵌小头像 + `transition`/`animation` 滑入；拖拽可用 **pointer events + position 持久化**，拖入底区 **dock 收起** 与 App 行为一致。
